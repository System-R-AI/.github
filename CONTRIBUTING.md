# Contributing to System R

Thanks for your interest in contributing to System R.

## What's open for contributions

| Repo | Type | Contributions welcome |
|------|------|----------------------|
| [systemr-python](https://github.com/System-R-AI/systemr-python) | Python SDK | Bug fixes, documentation, examples |
| [demo-trading-agent](https://github.com/System-R-AI/demo-trading-agent) | Reference agent | New strategies, framework integrations, docs |

## How to contribute

1. **Check existing issues** — your idea may already be tracked
2. **Open an issue first** — describe what you'd like to change before writing code
3. **Fork and branch** — create a feature branch from `main`
4. **Keep changes focused** — one PR per logical change
5. **Test your changes** — run `pytest` before submitting
6. **Submit a PR** — reference the issue number in your description

## Code style

- Python: PEP 8, type hints where practical
- Commit messages: imperative mood, concise ("Add retry logic to client", not "Added retry logic")

## Questions?

- Documentation: [docs.systemr.ai](https://docs.systemr.ai)
- Issues: [GitHub Issues](https://github.com/System-R-AI/systemr-python/issues)
